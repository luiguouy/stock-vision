# 模块调用关系与依赖图

> 本文档记录 StockVision 智能股票技术分析平台中各模块、函数、接口之间的调用关系与依赖方向，帮助 AI 理解"改了 A 会影响 B 和 C"。

---

## 1. 整体架构依赖方向

```
┌─────────────────────────────────────────────────────────┐
│                    前端 (Vue 3 SPA)                      │
│                                                         │
│  App.vue ──→ api.ts ──→ (fetch) ──→ Vite proxy ──→ FastAPI │
│    │            │                                      │
│    │            └──→ mockEngine.ts (降级本地计算)         │
│    │                                                    │
│    └──→ chartUtils.ts (图表配置)                         │
│                                                         │
├─────────────────────── Vite 代理边界 ─────────────────────┤
│                                                         │
│                    后端 (Python FastAPI)                  │
│                                                         │
│  main.py ──→ analysis.py (支撑阻力位 + 区间统计)          │
│    │                                                    │
│    └──→ data_provider.py (多源数据拉取 + 故障转移)        │
│              ├── TencentProvider (腾讯证券)               │
│              └── YahooProvider  (Yahoo Finance)          │
│                                                         │
├─────────────────────── 外部数据源 ────────────────────────┤
│  腾讯证券 API (web.ifzq.gtimg.cn)                        │
│  Yahoo Finance (yfinance 库)                              │
└─────────────────────────────────────────────────────────┘
```

**依赖方向原则：** 上层调用下层，反向依赖不存在。如果下层改了签名/返回值，必须同步更新上层调用方。

## 2. 后端模块调用关系

### 2.1 main.py 调用链

```
main.py
├── import analysis.calculate_sr_levels        ← 支撑阻力位计算
├── import analysis.calculate_range_stats      ← 区间涨跌统计
├── import data_provider.provider_manager      ← 全局数据源单例
├── import data_provider.SUPPORTED_PERIODS     ← 周期白名单
│
├── GET  /api/search(keyword?)
│   └── 纯本地逻辑，操作 STOCKS_POOL 列表
│
├── GET  /api/klines(symbol, period)
│   ├── 校验 period ∈ SUPPORTED_PERIODS
│   └── get_klines_df(symbol, period, full=True)  ← 含本地 SQLite 缓存 (TTL 24h)
│       └── 内部路由 → TencentProvider 或 YahooProvider
│
├── POST /api/analysis(AnalysisRequest)
│   ├── get_klines_df(symbol, period, full=True)
│   ├── calculate_sr_levels(df, period=period)
│   └── calculate_range_stats(df, start_date, end_date)
│
├── GET  /api/health
│   └── 探测后端存活，前端据此自动决定演示/实时模式（不依赖数据源）
│
└── GET  /api/periods
    └── 返回 SUPPORTED_PERIODS + 描述文本
```

**影响分析：**
- 修改 `analysis.py` 函数签名 → 必须同步更新 `main.py` 中 `analyze_stock()` 的调用
- 修改 `data_provider.py` 的 `SUPPORTED_PERIODS` → 影响 `main.py` 的周期校验逻辑和 `/api/periods` 返回
- 修改 `KLinePoint` / `AnalysisResponse` 等 Pydantic 模型 → 前端 TypeScript 接口必须同步

### 2.2 data_provider.py 内部调用关系

```
DataProviderManager (全局单例 provider_manager)
├── .fetch_klines(symbol, period, full)
│   ├── 判断 period → 决定主备源顺序
│   │   ├── '4h' → Yahoo 优先, Tencent 备选
│   │   ├── '1d'/'1w'/'1M' → Tencent 优先, Yahoo 备选
│   │   └── '3M'/'6M'/'1Y' → Tencent 日线优先, Yahoo 1d 备选 (聚合周期)
│   ├── primary.fetch_klines(symbol, period, full)
│   ├── (失败时) backup.fetch_klines(symbol, period, full)
│   └── (聚合周期) 主备都经 validate_klines(df,'1d') 校验 → 选"有效且根数更多"的一份
│
├── TencentProvider
│   ├── .fetch_klines(symbol, period, full)
│   │   ├── ._get_symbol_candidates(symbol) → 生成候选腾讯代码
│   │   ├── ._fetch_raw(candidate, tencent_period, fetch_count)
│   │   │   └── httpx.AsyncClient → 腾讯 API
│   │   ├── (聚合周期 3M/6M/1Y) validate_klines(df,'1d') → apply_forward_adjustment → ._resample_to_period(df, rule)
│   │   └── (need_resample=4h 时) ._resample_to_4h(df)
│   └── .supports_period(period)
│
└── YahooProvider
    ├── .fetch_klines(symbol, period, full)
    │   ├── ._to_yahoo_ticker(symbol) → 转换为 Yahoo ticker
    │   ├── asyncio.to_thread(._fetch_yfinance_sync, ..., resample_rule)
    │   │   ├── yf.Ticker(ticker).history(period, interval)
    │   │   └── (聚合周期) ._dispatch_resample → validate_klines(df,'1d') → 前复权 → _resample_to_period
    │   │   └── (4h) TencentProvider._resample_to_4h(df)
    │   └── ._calc_yahoo_period(interval, count) → 计算 Yahoo period 参数
    └── .supports_period(period)
```

**关键耦合点：**
- `YahooProvider._fetch_yfinance_sync` 复用了 `TencentProvider._resample_to_4h()` 与 `_resample_to_period()` 静态方法 — 修改重采样逻辑会同时影响两个数据源
- `PERIOD_MAP` 是周期映射的核心配置（`resample` 字段指向 `QS`/`2QS`/`YS`），修改它会影响所有聚合周期的行为
- 聚合周期（3M/6M/1Y）的前复权必须在 `_resample_to_period` **之前**对日线完成，Manager 不再二次复权

### 2.3 analysis.py 内部调用关系

```
analysis.py
├── PERIOD_PARAMS (常量表)
│   └── 被 calculate_sr_levels() 读取
│
├── calculate_sr_levels(df, window, distance_pct, period)
│   ├── 读取 PERIOD_PARAMS[period] → half_life, window
│   ├── scipy.signal.argrelextrema → 寻找局部极值
│   ├── 自适应聚类合并 (ATR 近似阈值)
│   ├── 时间衰减权重累加
│   └── 临近度打分 → 输出 Top3 支撑 + Top3 阻力
│
└── calculate_range_stats(df, start_date, end_date)
    ├── 筛选日期区间子集
    ├── 寻找最低点 → 之后最高点 → 计算最大上涨
    └── 寻找最高点 → 之后最低点 → 计算最大下跌
```

### 3.2 api.ts 内部调用关系

```
api.ts
├── searchStocks(keyword)
│   └── fetch GET /api/search
│
├── getKLines(symbol, period)
│   ├── fetch GET /api/klines
│   └── (失败 + 非4h周期) 降级: 直连腾讯 API

│
```

## 4. 前后端数据模型映射

| 后端 Pydantic 模型 | 前端 TypeScript 接口 | 定义位置 |
|---|---|---|
| `KLinePoint` | `KLinePoint` | main.py / api.ts + mockEngine.ts |
| `SRLevel` | `SRLevel` | main.py / api.ts + mockEngine.ts |
| `RangeStat` | `RangeStatPoint` | main.py / api.ts |
| `AnalysisResponse` | `AnalysisResponse` | main.py / api.ts |
| `AnalysisRequest` | (JSON body 内联) | main.py / api.ts |
| `StockInfo` (dict) | `StockInfo` | main.py (dict) / api.ts |

**注意：** `KLinePoint` 和 `SRLevel` 在 `api.ts` 和 `mockEngine.ts` 中各定义了一份，字段相同但是独立类型。修改时需两处同步。

## 5. 变更影响速查表

| 如果你要改... | 还需要同步检查/更新... |
|---|---|
| `analysis.py` 的 `calculate_sr_levels()` 签名或返回值结构 | `main.py` 的 `analyze_stock()`；前端 `mockEngine.ts` 的 `calculateSRLevels()` 保持算法一致 |
| `analysis.py` 的 `calculate_range_stats()` 返回值结构 | `main.py` 的 `analyze_stock()`；前端 `mockEngine.ts` 的 `calculateRangeStats()` |
| `analysis.py` 的 `PERIOD_PARAMS` | 前端 `mockEngine.ts` 的同名常量 `PERIOD_PARAMS` |
| `data_provider.py` 的 `SUPPORTED_PERIODS` | `main.py` 的周期校验；前端 `api.ts` 的 `SUPPORTED_PERIODS` |
| `data_provider.py` 的 `PERIOD_MAP` | 所有数据源的周期映射逻辑（含 `resample` 频率码） |
| `data_provider.py` 的 `_resample_to_period()` | 3M/6M/1Y 聚合周期都依赖它；维护时注意前复权顺序与 `QS/2QS/YS` 频率码 |
| `data_provider.py` 的 `_resample_to_4h()` | TencentProvider 和 YahooProvider 的 4h 数据都会受影响 |
| `data_provider.py` 的 `validate_klines()` | 改动粒度阈值会影响聚合周期的数据源选择与缓存校验 |
| `main.py` 的 Pydantic 模型字段 | 前端 `api.ts` 和 `mockEngine.ts` 中对应的 TypeScript 接口 |
| `api.ts` 的函数签名 | `App.vue` 中所有调用该函数的位置 |
| `mockEngine.ts` 的算法 | 后端 `analysis.py` 应保持算法一致（前后端对等降级） |
| `chartUtils.ts` 的 `getChartOptions()` | `App.vue` 的 `initChart()` 和 `toggleDarkMode()` |
| 前端 `KLinePoint` 接口字段 | 后端 `KLinePoint` Pydantic 模型；`mockEngine.ts` 的同名接口 |

## 7. 全局单例与共享状态

| 单例/共享 | 位置 | 用途 | 注意事项 |
|---|---|---|---|
| `provider_manager` | data_provider.py L414 | 全局数据源管理器 | 被 main.py 直接 import 使用 |
| `STOCKS_POOL` | main.py L48-59 | 本地美股股票池 | 搜索功能的唯一数据源 |
| `PERIOD_PARAMS` | analysis.py L28-33 | 多周期算法参数 | 前端 mockEngine.ts 有镜像副本（含 3M/6M/1Y） |
| `PERIOD_MAP` | data_provider.py L27-32 | 周期→数据源映射 | 含 `resample` 频率码（QS/2QS/YS）；修改影响所有数据源 |
| `PREWARM_PERIODS` | main.py | 启动预热周期列表 | 含 1d/1w/1M/3M/6M/1Y，后台预热热门股缓存 |
| `klineCache` | App.vue L631 | 前端 K 线缓存 (Map) | 按 `symbol_period` 键缓存 |
| `analysisCache` | App.vue L632 | 前端分析结果缓存 (Map) | 按 `symbol_period_startDate_endDate` 键缓存（含起止日期，v0.3.0 修复"改日期不重算"） |
| `watchlistKlineCache` | App.vue | 自选股日线缓存 (Map) | 键 `mock_${sym}`(演示) / `real_${sym}_1d`(真实)；与图表 `klineCache` 独立，按日线缓存供自选栏实时价计算（见技术债第 18 条） |
