## [0.4.0] - 2026-07-16

本期是一次跨市场能力大升级：打通美股 / A股 / 港股三市场（代码、搜索、行情、自选），新增「多市场实时行情」与「全局证券搜索（命令面板）」，并把自选股从扁平列表重构为专业终端式的「分组 + 居中 Modal」交互；同时修复了港股 / A股 K 线 404、A股/港股错路由到 Yahoo、全局搜索仅限美股池等多处缺陷。

### 新增
- **多市场实时行情（美股 / A股 / 港股）**：后端新增 `GET /api/quote` + `GET /api/quote_config`（腾讯公开行情源 `qt.gtimg.cn/q=`），内置服务端限流护盾——单标的最低刷新间隔、单 IP 滑动窗口计数（超额自动降速）、限流 / 空数据 / 超时后全局冷却 30s 再重试，从根上避免被数据源封禁。前端 `utils/quote.ts` 实现「分层刷新」引擎：本地定时器按各市场合规最低间隔轮询、价格与涨跌幅均未变化则不去重不重渲染、连续无新行情的冷门标的自适应降频、美股按美东时间区分盘前 / 盘中 / 盘后三套刷新配置；受限 / 冷却时进入冷却态而非高频重试。
- **全局证券搜索（命令面板式）**：新增 `components/GlobalSecuritySearch.vue`，常驻头部搜索框 + `Ctrl/⌘K` 全局唤起；支持最近搜索历史（可单条删除 / 清空）、热门推荐、键盘上下导航回车进入、♥ 一键加入 / 取消自选；后端 `/api/search` 升级为全市场五维匹配（代码 / 中英文名 / 拼音缩写 / 全拼），本地池未命中走腾讯全量证券搜索兜底，任意证券可定位。
- **自选股分组功能重构（专业终端式交互）**：从旧版「多选数组 + 下拉内联编辑」重构为「单选高亮分组 + 居中 Modal」——侧边栏顶部平铺 `全部` / `我的特选` Tab 与分组切换下拉；`＋ 创建分组` / `📁 分组管理` / `添加自选` 三个居中 Modal（磨砂玻璃阴影）分别处理建组、重命名 / 删除、向空分组搜索加股；空白分组显示空状态插画 + 一键添加，解耦了侧边栏的拥挤与保存冲突。后端 `/api/watchlist` 数据模型由扁平 `symbols` 升级为 `groups[{id,name,stocks}]`，旧版扁平 list 自动迁移进默认分组。
- **自选股列表全量显示价格与区间涨跌幅**：左侧自选股面板不再仅当前股票显示价格/涨跌幅（其余显示 `--`），现在**所有自选股均显示**各自在当前周期下的最新价与涨跌幅（日K=当日涨跌、周K=本周涨跌，随周期与结束日期同步更新）。切换日期范围时所有股票价格同步更新；真实模式专用 `watchlistKlineCache` 缓存避免切换日期时重复请求。

### 修复
- **港股 / A股 K 线 404（严重）**：腾讯 k 线接口对市场前缀大小写敏感（`HK00700` / `SH600519` 直接返回 `param error`），前端与 `/api/watchlist` 统一返回大写前缀导致港股 / A股全部 404。新增 `norm_market_prefix` 在 `get_klines_df` 入口统一规范为小写，并修正 `_get_symbol_candidates` 对 sh/sz/hk 候选转小写返回；缓存键同步归一化。美股分支因自带小写 `us` 前缀不受影响。
- **A股 / 港股被错路由到 Yahoo（取数不稳定 / 取不到）**：`fetch_klines` 此前对非美股也走 Yahoo(主) / 腾讯(备)，但 Yahoo 对 A股 / 港股覆盖差。改为非美股仅走腾讯证券（原生支持 sh/sz/hk），美股仍 Yahoo 优先 + 腾讯兜底。
- **Yahoo ticker 映射错误（A股 / 港股）**：`_to_yahoo_ticker` 此前未处理 sh/sz/hk，兜底时生成的 ticker 不正确。补 `sh → .SS` / `sz → .SZ` / `hk → 5 位 .HK` 映射。
- **全局搜索仅限美股池（中文名 / 拼音 / 港股 / A股都搜不到）**：旧 `/api/search` 只在本地美股池匹配、纯字母硬拼 `us` 前缀。改为全市场五维匹配，本地池未命中走腾讯全量搜索兜底；自然语言无果直接返回空，不再生成 `US台积电` 这类假代码。
- **自选股列表初始加载崩溃（白屏，严重）**：`loadWatchlist` 异步完成即渲染，但 `updateWatchlistPrices` 要等 `loadStock` 完成，此间 `watchlistPriceMap` 为空 `{}`；模板守卫 `price !== null` 挡不住 `undefined`，导致 `undefined.toFixed()` 抛 TypeError 白屏——**每次初始加载必现**。已将守卫改为 `!= null`，并在 `loadWatchlist` 设好自选股后初始化 priceMap 为全 loading 态。
- **自选股涨跌幅不随 K 线周期变化**：`calcPriceInRange` 原计算区间起止 close 累计涨跌，当范围为"全部"时日K/周K 起止相同、切换周期不变。改为取区间末尾那根 close vs 前一根 close——日K 当日、周K 本周、月K 本月涨跌，切换周期即时变化。
- **自选股初始加载竞态导致永卡 loading 骨架屏**：`loadWatchlist` 与 `loadStock` 并发，`loadStock` 先完成时 `updateWatchlistPrices` 因 `watchlist` 为空而 return，后续无人再触发更新。已在 `loadWatchlist` 设好自选股后补调更新（若 `loadStock` 已完成）。
- **真实模式下自选股列表非当前股票全部 404（严重）**：`updateWatchlistPrices` 用原始代码（如 `"AAPL"`）请求，缺 `us` 前缀导致后端 404。已在真实模式分支补 `us` 前缀并与 `cacheKey` 保持一致。
- **测试模式下自选股 mock 数据每次刷新都变**：每次调用都重新 `generateMockKLines` 导致价格随机跳。已改测试模式也用 `watchlistKlineCache`（key 加 `mock_` 前缀），每 `(symbol, period)` 只生成一次。
- **切换测试 / 真实模式后自选股仍显示旧 mock 数据（数据混合）**：模式切换 watcher 只重载当前股票，未清空自选股缓存。已先 `clear` `watchlistKlineCache` 与 `watchlistPriceMap` 再重载，确保所有自选股用新数据源重算。

### 优化
- **后端启动方式升级**：用 `lifespan`（`@asynccontextmanager`）替代已废弃的 `@app.on_event("startup")`，初始化与预热更规范。
- **新增 `GET /api/health` 轻量健康检查**：前端据此自动判定演示 / 实时模式，不依赖任何外部数据源。
- **股票池扩展至多市场**：本地常用池由纯美股扩至沪A（5）/ 深A（5）/ 港股（5），预热 `PREWARM_SYMBOLS` 同步覆盖多市场，首次打开即命中缓存。

### 文档
- 新增 `多市场实时行情_改动说明.md`；同步更新项目结构梳理 / 数据模型与接口契约 / 模块调用关系 / 降级策略 / 技术债 / 进展清单等多份项目文档。

## [0.3.0] - 2026-07-09

围绕「区间统计准确性、多周期渲染、测试模式体验」的一次修复版本：区间统计改用最大回撤 / 反弹算法并修复缓存失效，测试模式补齐各周期重采样，并修复短历史股票（重新上市的 SNDK）在大周期下的报错 / 不显示问题。

### 新增
- **区间选择按股票记忆**：每支股票的起始/结束日期写入 `localStorage`（`stockvision_range_prefs`，按股票代码分键），切换股票再切回时自动恢复该股票上次用过的区间；越界自动 clamp 到该股票 K 线范围内。
- **自定义丝滑日期选择器**（替换原生日历）：新增 `DateRangePicker.vue` 组件，弹层日历支持**年份/月份下拉快速跳转**（免去原生日历逐月滚动的"涩感"）、范围高亮、越界禁用、`fade-scale` 平滑过渡；选完起始自动切到结束字段。
- **区间快捷预设**：区间统计面板新增 `近1周 / 近1月 / 近3月 / 近6月 / 近1年 / 全部` 一键按钮（新增"近1周"），点选即设定区间并重新计算，基本不再需要手动抠日历。

### 修复
- **周K/月K/季K/半年K/年K 在测试模式下"显示成了日K"**：默认开启的测试模式用 `generateMockKLines` 只生成 4500 根日线，此前**完全没有按周期重采样**，所以无论选哪个周期都把日线直接渲染出来，视觉上"周K=月K=日K"。已在 `mockEngine.ts` 新增 `resampleMockKLines`（按周一归桶→周 / 按月 / 按季 / 按半年 / 按年聚合，open=首根、high=max、low=min、close=末根、volume=求和），测试模式生成日线后立即按 `currentPeriod` 重采样，现在各周期显示正确。真实数据模式不受影响（后端本就返回正确粒度）。
- **区间统计改日期后"重新计算"无反应**：实时数据模式下，前端 `analysisCache` 的缓存键原本只含 `symbol+period`、不含起止日期，导致改了起始/结束日期后命中旧缓存、返回旧区间的统计（面板看起来没变化）。已将日期区间纳入缓存键，改日期即重新计算。
- **最大连贯下跌/上涨不随日期区间变化**：区间统计算法原采用"取区间内全局最高点、再找其后的最低点"计算最大下跌，当区间最高点落在末尾（近期上涨行情极常见）时结果退化为 0 且不随区间改变；同法的最大上涨在下跌行情也存在类似隐患。已改为全程跟踪"运行峰值 / 运行谷值"的最大回撤 / 最大反弹算法（前端 `mockEngine` 与后端 `analysis` 同步修正），现在任一区间都能正确捕获真实的最大连贯涨跌。
- **短历史股票（重新上市的 SNDK）在季K/半年K/年K 周期下报错/不显示**：SNDK 重新上市仅约 1.4 年日线，重采样后半年K~3 根、年K~2 根，旧 `validate_klines` 统一要求 ≥5 根被拒。已按周期设置差异化最小根数门槛（日K≥5 / 周K≥3 / 月K≥2 / 聚合周期≥1），并把聚合周期的粒度校验从"单纯上限"改为**区间带**（6M: 100<间隔≤260、1Y: 260<间隔≤400），彻底堵住"季度桩(90天)冒充半年/年线"的漏洞；前端对 ≤10 根短数据集一律 `fitContent` 全量展示，避免长股票留下的高比例缩放把仅有的几根 K 线推到屏幕外。后端进程修改后已重启加载新代码。

## [0.2.0] - 2026-07-09

围绕「数据完整性、多周期、加载速度」的一次大版本更新：K 线从"最近约 1.3 年"升级为全量历史、新增季/半年/年 三种大周期、引入本地缓存与前复权，并系统性修复了时区、粒度、缩放等多个数据对应问题。

### 新增
- **全量历史 K 线**：后端 `fetch_klines` 新增 `full` 参数，`/api/klines`、`/api/analysis` 改为全量拉取；日 K 不再截断为最近约 320 根，可显示上市至今。
- **季 K / 半年 K / 年 K 三种真实周期**（3M / 6M / 1Y）：每根蜡烛分别聚合为一个季度 / 半年 / 一年，由日线按日历时间重采样合成，支撑阻力位随周期自适应。
- **SQLite 本地缓存层**（`db_cache.py` + `stock_data.db`）：拉取过的全量历史落地本地，二次加载 / 刷新 / 重启后毫秒级返回；带 24 小时 TTL。
- **启动预热**：后台预热热门股（AAPL/NVDA/TSLA/AMD/MSFT/META/GOOGL/AMZN/MU）× 多周期，首次打开页面即命中缓存。
- **前复权（拆股复权）**：`apply_forward_adjustment` 自动识别拆股比例并复权历史价格，消除拆股导致的价格断层。
- **自选股后端持久化**：新增 `GET/POST /api/watchlist`，以 `watchlist.json` 存储；前端优先读后端、localStorage 兜底，解决跨预览 / 跨 origin 自选股丢失。
- **K 线图缩放记忆**：鼠标滚轮缩放后，切换日期范围或股票时保持缩放比例（持久化到 localStorage）。
- **骨架屏加载动画**：仿真实图表的扫光骨架屏（56 根 K 线柱 + 量能 + 价格轴占位），支持浅色 / 深色双主题，最少展示 700ms 保证动画可感知。
- **数据完整性提示**：真实模式日 K 返回 < 400 根时，图表上方显示琥珀色警告条，避免残缺数据误导。
- **数据粒度校验守卫** `validate_klines`：按相邻日期中位数间隔判断粒度，拒绝被代理退化的"季度桩 / 年聚合"假数据。

### 修复
- **K 线数据不完整**：移除写死的 `count=320` 与腾讯源截断逻辑，日 K 不再只显示最近约 1.3 年。
- **重采样标签错位**（季/半年/年 K "每根柱子对应不对"）：重采样频率码从已废弃的 `Q/2Q/YE` 改为起始日锚定的 `QS/2QS/YS`，消除前导空桶和"季末而非季初"的标签错位。
- **前复权顺序错误**：前复权必须在聚合重采样"之前"对日线完成，否则大周期会重现拆股缺口。
- **时区导致的日期整体偏移**：Mock 数据改用本地年月日格式化（不再 `toISOString` 回退一天）；后端 Yahoo 直连与 yfinance 兜底统一用美东时区（`America/New_York`）输出交易日；腾讯日期经 `_normalize_date` 规范。
- **真实模式起点停在 2025-03-28**：删除前端 `getKLines` 里"静默直连腾讯（仅约 320 根）却伪装成完整数据"的降级路径；前端显式传 `full=true`，后端失败直接抛明确错误。
- **每根 K 线跨 3 个月（粒度错位）**：识别并拒绝沙箱 Yahoo 代理返回的季度桩数据；缓存读取后二次校验粒度，污染缓存视为未命中重拉。
- **自选股丢失**："上次加了这次没了"改为后端 JSON 持久化，跨预览 / 跨 origin 生效。
- **缩放记忆与日期范围冲突**：`setTimeRange` 改用柱子索引 + `setVisibleLogicalRange` 精确定位，并同步更新缩放记忆，避免后续数据变化用旧比例覆盖用户选择。
- **构建阻断的 TS 错误**：移除 lightweight-charts v5 已废弃的 `watermark` 选项及未使用变量，`vue-tsc` / `vite build` 恢复通过。
- **腾讯 full 模式取数失败**：`count` 由 100000 降为 2000（腾讯单页上限），避免返回 `param error` 空数据。

### 优化
- **数据源双源择优**：主备两源都做粒度校验，选"有效且根数更多"的一份；两份都无效则返回 404，不再静默返回错误数据。
- **多数据源自动故障转移**：全量优先 Yahoo 直连 chart API（`range=max`，比 yfinance 稳）→ yfinance → 腾讯兜底。
- **setData 前排序去重**：按时间严格升序并去重，防御图表乱序 / 重复渲染错位。
- Mock 模式历史天数增至约 18 年，沙箱内也能直观看到长历史。

### 文档
- 同步 `project-structure-tracker` 生成的 8 份项目文档（概览、数据契约、核心算法、降级策略、技术债、进展清单、模块依赖、结构梳理），首次纳入版本库。

### 工程
- `.gitignore` 屏蔽运行时产物（`stock_data.db` / `backend.log` / `watchlist.json`）。
- 新增离线单元测试：`test_resample.py`（重采样 5 项）、`test_validate.py`（粒度校验）、`test_adjustment.py`（前复权 12 项）。
