# 前端测试模式使用说明

## 🎯 功能概述

为了方便前端独立开发和调试，我添加了**测试数据模式**，让你无需启动后端服务即可查看页面布局和交互效果。

## ✨ 主要特性

### 1. **测试模式开关**
- 位置：页面右上角（头部导航栏）
- 默认状态：**开启**（🧪 测试模式）
- 切换方式：点击开关即可在"测试模式"和"真实数据"之间切换

### 2. **模拟K线数据生成**
当测试模式开启时，系统会：
- ✅ 自动生成逼真的日K线数据（320个交易日）
- ✅ 根据不同股票设置不同的波动率（如 TSLA 波动更大，MSFT 更稳定）
- ✅ 自动跳过周末，符合真实交易日历
- ✅ 使用随机游走算法，每次生成的数据略有不同

### 3. **本地技术分析计算**
- ✅ 支撑/阻力位自动计算（基于前端 `mockEngine.ts`）
- ✅ 区间涨跌统计自动计算
- ✅ 图表标记自动绘制

## 🚀 使用步骤

### 方式一：纯前端调试（推荐）

```bash
# 1. 进入前端目录
cd stage_one/frontend

# 2. 安装依赖（如果还没安装）
npm install

# 3. 启动开发服务器
npm run dev

# 4. 打开浏览器访问 http://localhost:5173
```

**此时无需启动后端！** 直接在搜索框输入股票代码（如 AAPL、NVDA、TSLA），即可看到：
- 📊 K线图正常渲染
-  支撑/阻力线自动显示
-  区间涨跌统计数据
- ⭐ 自选股管理功能

### 方式二：切换到真实数据

如果你已经启动了后端服务（端口 8000），可以：

1. 点击右上角的开关，切换到 **🌐 真实数据** 模式
2. 重新搜索股票代码
3. 系统会从腾讯证券接口获取真实的K线数据

## 🔧 技术实现细节

### 数据生成算法
```typescript
// 随机游走模型
price[t] = price[t-1] * (1 + random_change)
random_change ∈ [-volatility, +volatility]

// 高低点计算
high = max(open, close) * (1 + random_factor)
low = min(open, close) * (1 - random_factor)
```

### 波动率映射
每只股票都有预设的波动率参数，模拟真实市场特征：
- 低波动 (< 2.5%)：AAPL, MSFT
- 中波动 (2.5%-3.5%)：GOOGL, AMZN, META, SNDK
- 高波动 (> 3.5%)：NVDA, TSLA, AMD, MU

## 💡 注意事项

1. **每次刷新页面数据会重新生成**：因为使用随机算法，所以每次加载的K线图略有不同
2. **测试数据不持久化**：不会保存到 localStorage 或数据库
3. **不影响后端逻辑**：测试模式完全在前端运行，与后端无关
4. **可随时切换**：点击右上角开关即可无缝切换到真实API

##  故障排查

### 问题1：图表不显示
- 检查控制台是否有报错
- 确认 lightweight-charts 已正确安装
- 尝试刷新页面

### 问题2：分析结果异常
- 检查日期范围是否合理（不能跨度过大）
- 查看控制台日志，确认 `calculateSRLevels` 是否正常执行

### 问题3：想关闭测试模式但后端没启动
- 先启动后端：`cd stage_one/backend && python main.py`
- 或者保持测试模式开启，继续前端开发
