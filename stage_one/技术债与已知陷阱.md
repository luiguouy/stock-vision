# 技术债与已知陷阱

> 最后更新：2026-07-15 22:00

## App.vue 超大单文件架构（God Component）

**严重程度**：高
**位置**：`frontend/src/App.vue`（2698 行）

整个前端应用——模板（~720 行 HTML）、脚本（~1750 行 TypeScript）、样式——全部塞在单个 Vue SFC 中。涵盖导航栏、搜索、图表、自选分组 CRUD、实时行情集成、暗色模式、分析面板、区间统计、拖拽调宽、tooltip、缩放记忆、键盘快捷键等全部功能。

**影响范围**：任何 UI 修改都需要在 2698 行中定位上下文，编辑窗口大、token 消耗高，误改风险大。新增功能会让文件继续膨胀。
**来龙去脉**：有意为之 — 项目定位为 stage_one 快速迭代原型，单文件避免了组件间通信的复杂度，但已接近可维护性上限。

**改进建议**：拆分为 NavBar、WatchlistSidebar、ChartPanel、AnalysisPanel、StatusBar 等独立组件，每个控制在 300-500 行。但需注意 props/emit 传递链的设计，避免过度碎片化。建议在 stage_two 执行。

## data_provider.py 重复 except 块（死代码）

**严重程度**：中
**位置**：`backend/data_provider.py`（约第 503-508 行）

`_resample_to_period()` 方法中存在两个完全相同的 `except Exception` 块，第二个永远不可达。这是复制粘贴错误。

**影响范围**：运行时不会崩溃（Python 只执行第一个 except），但掩盖了可能的意图——第二个可能原本想捕获不同类型的异常。
**来龙去脉**：临时妥协 — 快速开发时的复制粘贴遗留。

**改进建议**：删除第二个重复的 except 块。

## SSL 证书验证被全局关闭

**严重程度**：中
**位置**：`backend/quote_provider.py`（约第 135 行）

`httpx.AsyncClient` 创建时设置了 `verify=False`，注释说是为了兼容沙箱/内网自签证书。但这意味着生产环境也跳过了 SSL 验证。

**影响范围**：安全风险——中间人攻击无法被检测。
**来龙去脉**：有意为之 — 开发环境需要绕过证书问题，但忘了加环境条件判断。

**改进建议**：改为 `verify=os.getenv("SSL_VERIFY", "true").lower() != "false"`，默认开启验证。

## print() 替代 logging 模块

**严重程度**：中
**位置**：`backend/` 全部 6 个 .py 文件（共 16+ 处）

整个后端使用 `print()` 输出诊断信息，未使用 Python `logging` 模块。没有日志级别、结构化输出、日志轮转。`backend.log` 文件存在，说明用了 ad-hoc 的重定向方式。

**影响范围**：生产环境无法按级别过滤日志、无法重定向到不同输出、无法控制日志大小。
**来龙去脉**：临时妥协 — 原型阶段用 print 快速调试，未迁移到正式日志系统。

**改进建议**：引入 `logging` 模块，按模块配置 logger name，关键错误用 `logging.exception()` 保留堆栈。

## 宽泛 except Exception 静默吞错

**严重程度**：中
**位置**：`backend/data_provider.py`（8 处）、`backend/analysis.py`（2 处）、`backend/main.py`（3 处）、`backend/quote_provider.py`（3 处）

多处 `except Exception` 捕获后静默返回空结果，甚至不打印错误信息（如 `data_provider.py` 第 98 行的 "保守放行"、`main.py` 第 539 行的 watchlist 加载异常）。

**影响范围**：生产环境出问题时极难排查——数据获取失败、分析异常都被静默吞掉了。
**来龙去脉**：有意为之 — 为了保证用户体验不中断（"宁可展示空数据也不弹错误"），但缺乏日志记录。

**改进建议**：至少在每个 except 块加 `logging.warning(..., exc_info=True)`，保留堆栈信息。

## 废弃的 @app.on_event("startup")

**严重程度**：中
**位置**：`backend/main.py`（第 306 行）

FastAPI 自 0.93.0 起已废弃 `@app.on_event()`，推荐使用 `lifespan` 上下文管理器模式。当前写法会产生废弃警告，未来 FastAPI 版本可能移除。

**影响范围**：升级到新版 FastAPI 时可能报错。
**来龙去脉**：临时妥协 — 项目启动时使用的旧 API，后续未迁移。

**改进建议**：迁移到 `lifespan` 模式：`@asynccontextmanager async def lifespan(app): ...`。

## User-Agent 字符串三处重复

**严重程度**：中
**位置**：`backend/data_provider.py`（第 253、633 行）、`backend/quote_provider.py`（第 32-34 行）

同一个 User-Agent 字符串在三个地方独立定义。`quote_provider.py` 提取为模块常量 `_USER_AGENT`，但 `data_provider.py` 在两处内联定义。

**影响范围**：更新 UA 时需改三处，容易遗漏。
**来龙去脉**：临时妥协 — 各模块独立开发时各自定义。

**改进建议**：提取到公共模块（如新建 `config.py` 或放入 `quote_config.py`），各模块统一引用。

## 前后端 SR 算法双实现维护成本

**严重程度**：中
**位置**：`backend/analysis.py` + `frontend/src/utils/mockEngine.ts`

支撑/阻力位算法在 Python 和 TypeScript 中各实现一份。`mockEngine.ts` 头部注释明确标注"移植自后端 analysis.py"。`PERIOD_PARAMS` 表必须手动保持 1:1 同步。

**影响范围**：后端算法变更时必须同步修改前端，否则降级模式输出结果不一致。
**来龙去脉**：有意为之 — 这是降级设计的核心——后端不可用时前端自包含运行。双实现是架构决策，不是疏忽。

**改进建议**：不要消除双实现（这是设计意图），但应在 `PERIOD_PARAMS` 等关键参数旁加注释提醒"修改此处需同步 mockEngine.ts"。考虑在 CI 中添加参数一致性检查。

## 数据文件与源码同目录

**严重程度**：中
**位置**：`backend/main.py`（第 510 行）、`backend/db_cache.py`（第 26 行）

`watchlist.json` 和 `stock_data.db` 存储在 backend 源码目录中。源码目录必须可写，重新部署时数据可能丢失。

**影响范围**：部署灵活性受限——源码目录和数据目录耦合。
**来龙去脉**：临时妥协 — 原型阶段图方便，未做数据目录分离。

**改进建议**：使用环境变量 `DATA_DIR` 指定数据目录，默认 `~/.stockvision/data/`。

## 每次请求新建 httpx.AsyncClient

**严重程度**：低
**位置**：`backend/data_provider.py`（第 255、638 行）

`TencentProvider` 和 `YahooProvider` 每次请求都 `async with httpx.AsyncClient() as client:`，不复用连接池。而 `quote_provider.py` 正确使用了模块级共享客户端。

**影响范围**：频繁请求时浪费 TCP 连接建立开销。
**来龙去脉**：临时妥协 — 各模块独立开发，未统一 HTTP 客户端策略。

**改进建议**：参照 `quote_provider.py` 的模式，在 `data_provider.py` 使用模块级共享客户端。

## AI 修改注意事项

- **修改 App.vue 任何区域时**：文件 2698 行，务必先用 Read 定位到精确区域再编辑，不要全量读取后凭记忆修改。template/script/style 三段各有明确分界。
- **修改 analysis.py 的 PERIOD_PARAMS 或聚类逻辑时**：必须同步修改 `frontend/src/utils/mockEngine.ts` 中对应实现，否则降级模式结果不一致。
- **`norm_market_prefix()` 函数**（main.py 第 147-156 行）看似多余（前端已经传了正确前缀），但实际是修复港股/A股 K 线 404 的关键——腾讯接口对前缀大小写敏感，必须在此处统一转小写。不要删除或"优化"掉。
- **`validate_klines()` 返回 True 的 except 块**（data_provider.py 第 98-100 行）看似是 bug，实际是有意设计——"保守放行，避免误杀正常数据"。不要改为返回 False。
- **data_provider.py 第 503-508 行的重复 except 块**是真正的死代码，可以安全删除第二个。
- **自选股 watchlist.json 格式**经历过从扁平 list 到分组 dict 的迁移（main.py 第 521-533 行），兼容旧格式的代码不要删除，除非确认所有用户已迁移。
- **`console.log` 残留在 App.vue 第 2298 行**——这是测试模式的调试日志，不是 bug，但生产环境应清理。
