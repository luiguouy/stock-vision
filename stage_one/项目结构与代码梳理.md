# StockVision 项目结构与代码梳理

> 最后更新：2026-07-15 22:00

## 项目概述

StockVision 是一个多市场（美股/A股/港股）智能股票技术分析平台，前端使用 Vue 3.5 + TypeScript + lightweight-charts 渲染 K 线图与交互界面，后端使用 FastAPI + pandas + scipy 提供数据获取、技术分析计算和实时行情聚合。系统支持 7 种 K 线周期（4h/1d/1w/1M/3M/6M/1Y），内置支撑/阻力位自动识别与区间涨跌幅统计，数据源采用腾讯证券（主）+ Yahoo Finance（备）双源故障转移，并实现多市场分层实时行情轮询与限流保护。

## 核心模块说明

### 1. App.vue — 前端单文件主组件

**职责**：承载全部 UI 渲染、状态管理、图表交互、导航历史、自选分组操作。

**关键内部结构**：
- 浏览器式导航栏（后退/前进/刷新 + 搜索栏 + 下拉建议），基于 `navBackHistory` / `navForwardHistory` 双栈
- 侧边栏：自选分组列表（搜索已移至 header）
- 主区域：lightweight-charts K 线图 + 工具栏（周期切换/指标开关）
- 右面板：支撑/阻力位列表 + 区间统计卡片
- 底部状态栏：浅色简约风格

**外部接口**：调用 `utils/api.ts` 的 `fetchKlines`、`fetchAnalysis`、`fetchQuote` 等函数与后端通信。

### 2. data_provider.py — 数据源抽象层

**职责**：统一封装腾讯证券和 Yahoo Finance 两个数据源，对上层提供透明故障转移的 K 线获取。

**核心类**：
- `DataProviderManager`（单例）：协调两个 Provider，对每个 (symbol, period) 分别尝试两个源，取通过验证且 bar 数最多的结果（best-of，非 first-success）
- `TencentProvider`：主源，擅长日/周/月线（直接拉取）
- `YahooProvider`：备源，擅长 4h 线（由 1h 重采样）和美股全量历史

**周期映射**：
| 内部代码 | 腾讯 | Yahoo |
|---------|------|-------|
| 4h | 不支持（Yahoo 1h→重采样） | 1h→resample 4h |
| 1d | 日K直接拉取 | 日线 |
| 1w | 周K直接拉取 | 日线→resample |
| 1M | 月K直接拉取 | 日线→resample |
| 3M/6M/1Y | 日线→resample | 日线→resample |

**对外接口**：`provider_manager.fetch_klines(symbol, period, full)` → `pd.DataFrame`

### 3. analysis.py — 技术分析核心

**职责**：计算支撑/阻力位（SR levels）和区间最大涨跌幅统计。

**SR 算法五步流程**：
1. 用 `scipy.signal.argrelextrema` 检测局部价格极值
2. 对极值点做时间衰减加权（half_life 随周期自适应）
3. 价格相近的极值聚类合并（ATR 自适应阈值）
4. 按聚类权重排序，取 Top-N 作为最终 SR 位
5. 标注类型（support/resistance）和强度（Weak/Medium/Strong）

**周期参数表**：
| 周期 | half_life | window |
|------|-----------|--------|
| 4h   | 30        | 5      |
| 1d   | 60        | 5      |
| 1w   | 26        | 4      |
| 1M   | 12        | 3      |

**对外接口**：`calculate_sr_levels(df, period)` → `List[dict]`，`calculate_range_stats(df, start, end)` → `dict`

### 4. quote_provider.py + quote_config.py — 实时行情

**职责**：通过腾讯 qt.gtimg.cn 免费接口获取多市场实时行情，内置三层限流保护。

**三层限流**：
1. 全局冷却（`RATE_LIMIT_COOLDOWN=30s`）：触发 403/429 后全量暂停
2. 单标的间隔（`ABSOLUTE_MIN_INTERVAL=0.1s`）：硬性合规红线
3. IP 滑动窗口（`IP_MAX_REQUESTS_PER_WINDOW`）：防单 IP 过载

**自适应机制**：连续多次无新行情的冷门标的，自动放大轮询间隔（`ADAPTIVE_COLD_THRESHOLD` → `ADAPTIVE_COLD_FACTOR` → `ADAPTIVE_MAX_FACTOR`）。

**配置集中**：所有数值在 `quote_config.py` 定义，前端通过 `/api/quote_config` 获取，不在前端硬编码。

**对外接口**：`GET /api/quote?symbols=usAAPL,sh600519` → `{quotes: [...], engine: {...}}`

### 5. db_cache.py — SQLite 缓存层

**职责**：将每个 (symbol, period) 的全量 K 线持久化到本地 SQLite，避免重复从外部拉取。

**表结构**：
- `klines` 表：(symbol, period, date) 为主键，存 OHLCV
- `meta` 表：记录每个 (symbol, period) 的最后更新时间与数据源

**读写流程**：读取时检查 TTL（默认 24h），过期则重新拉取后写回；写入时做数据校验（validate_klines），读取时再做一次校验（双重验证）。

**对外接口**：`get_cached_klines(symbol, period, max_age_hours)` → `List[dict] | None`，`save_klines(symbol, period, klines, source)`

### 6. api.ts — 前端 API 封装

**职责**：封装所有后端 API 调用，提供多市场工具函数和 TypeScript 类型定义。

**核心函数**：
- `fetchKlines(symbol, period)` → K 线数据
- `fetchAnalysis(symbol, startDate, endDate, period)` → SR + 区间统计
- `fetchQuote(symbols)` → 实时行情
- `searchStocks(keyword)` → 模糊搜索
- `fetchWatchlist()` / `saveWatchlist(groups)` → 自选分组

**市场工具函数**：`marketOf`、`stripPrefix`、`normalizeSymbol`、`formatSymbol` — 前后端共用同一套前缀规则（US/SH/SZ/HK）。

### 7. mockEngine.ts — 前端技术分析引擎

**职责**：纯前端计算版本的 SR 算法和区间统计，在后端不可用时降级使用。

**关键约束**：必须与后端 `analysis.py` 保持 1:1 算法同步，任何后端算法变更必须同步到此文件。

## 数据流与依赖关系

```
用户操作
  │
  ▼
App.vue ────────► api.ts ────────► FastAPI (main.py)
  │                  │                  │
  │                  │                  ├──► data_provider.py ──► 腾讯证券(主)
  │                  │                  │                      └─► Yahoo Finance(备)
  │                  │                  │
  │                  │                  ├──► db_cache.py ◄──── SQLite
  │                  │                  │
  │                  │                  ├──► analysis.py ──► scipy + pandas
  │                  │                  │
  │                  │                  ├──► quote_provider.py ──► 腾讯 qt 接口
  │                  │                  │         ▲
  │                  │                  │         │
  │                  │                  └──► quote_config.py (配置源)
  │                  │
  │                  └─── mockEngine.ts (降级计算)
  │
  ├──► chartUtils.ts ──► lightweight-charts
  │
  └──► quote.ts ──► /api/quote + /api/quote_config
```

**关键依赖链**：
1. **K 线数据链**：App.vue → api.ts → main.py → data_provider.py → (腾讯|Yahoo) → db_cache.py(缓存)
2. **分析计算链**：App.vue → api.ts → main.py → analysis.py → scipy/pandas
3. **实时行情链**：App.vue → quote.ts → /api/quote → quote_provider.py → 腾讯 qt 接口
4. **降级链路**：后端不可用时，App.vue → mockEngine.ts（纯前端计算 SR + 区间统计）
5. **配置下发链**：quote_config.py → /api/quote_config → quote.ts（前端不硬编码任何数值）

## 前后端约定

**统一代码格式**：`{市场前缀}{代码}`，如 `usAAPL`、`sh600519`、`sz000001`、`hk00700`。

**市场前缀映射**：
| 市场 | 前缀 | 示例 |
|------|------|------|
| 美股 | US | usAAPL |
| 沪A | SH | sh600519 |
| 深A | SZ | sz000001 |
| 港股 | HK | hk00700 |

**注意**：腾讯接口对市场前缀大小写敏感，内部统一大写，调用腾讯时 `norm_market_prefix()` 转为小写。
