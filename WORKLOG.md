# 工作日志 · WORKLOG

本项目所有开发工作的**双层记录**。

- 🔴 **大版本**：里程碑发布 → `vX.Y.0` annotated tag + GitHub Release + `CHANGELOG.md`（版本叙事、对外说明）
- 🟢 **小版本**：日常增量改动 → 轻量 tag `snap-YYYY-MM-DD[-N]` + 本文件一条记录（个人工作流水、便于回滚）

> 关系：小版本是日常锚点，积累到一定量级就打包成一个大版本。大版本发布时把对应区间的小版本工作收口进 `CHANGELOG.md` 的新版本段落。

## 使用约定
- 每次做了值得保留的小改动：打一个轻量 tag，并在下方加一条记录（**最新在上**）。
- tag 命名：`snap-2026-07-16`、`snap-2026-07-16-2` …（同一天多个加序号）。
- 推送到远程（让 tag 在 GitHub 可见、回滚有备份）：`git push origin snap-2026-07-16`
- 大版本发布流程见 `CHANGELOG.md` 顶部格式；发布后用 `git tag -a vX.Y.0 <commit> -m "..."` 打 annotated tag 并 `gh release create`。

## 回滚速查（三种安全级别，按需选）
1. **只看不改**（临时查看旧版本，不碰当前进度）：
   `git checkout snap-2026-07-16` → 看完 `git checkout main` 回来
2. **安全回滚（推荐，保留全部历史）**：
   生成反向 commit 抵消改动：`git revert <commit-hash>`
   或基于某快照开回滚分支慢慢改：`git checkout -b rb-test snap-2026-07-16`
3. **硬回滚（⚠️ 丢弃该点之后的所有历史，不可轻易恢复）**：
   `git reset --hard snap-2026-07-16`（仅在确认不要之后历史时用）
