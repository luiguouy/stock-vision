# 智能股票技术分析平台 — 项目结构与代码梳理

> **最后更新：2026-07-15 22:57**
> **项目性质**: 个人练习项目，基于 Web 的多市场（美股/A股/港股）技术分析平台
> **当前阶段**: 第一阶段 (MVP) — 多数据源 + 多周期 + 实时行情 + 自选分组
> **技术栈**: Vue 3.5 + TypeScript 6 + Vite 8 (前端) / Python + FastAPI 0.111 (后端)
> **图表库**: TradingView Lightweight Charts v5.2.0
> **数据源**: 腾讯证券 (主，覆盖全市场) + Yahoo Finance (美股全量优先) — 自动故障转移 + 本地 SQLite 缓存
> **实时行情**: 腾讯免费公开接口 `qt.gtimg.cn/q=` (美股/A股/港股，含限流护盾)
> **支持周期**: 日K / 周K / 月K / 季K(3M) / 半年K(6M) / 年K(1Y)；4h 后端保留但前端 UI 已隐藏
> **前端测试模式**: 支持纯前端 Mock 数据生成 + 本地分析计算，无需后端即可调试

## 二、后端代码详解 (`stage_one/backend/`)

### 2.1 `main.py` — FastAPI 主入口

**职责**: 提供 10 个路由（9 个 `/api` 端点 + 根重定向）+ 自选分组持久化 + 符号前缀归一。

**启动依赖**: `lifespan` 取代旧 `@app.on_event` — 启动 `init_db()` + 后台 `prewarm_cache()`（预热热门股 × 周期）；CORS 全开；根 `/` 重定向 `/docs`。

**10 个路由**:

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 重定向到 `/docs` |
| GET | `/api/health` | 存活探针，前端据此自动判定演示/实时模式 |
| GET | `/api/quote` | 批量多市场实时行情（腾讯 `qt.gtimg.cn/q=`） |
| GET | `/api/quote_config` | 下发行情刷新/限流/延迟全部配置 |
| GET | `/api/search` | 全市场单搜索（内置池 + 腾讯 smartbox 兜底） |
| GET | `/api/klines` | 多周期 K 线（全量历史，含本地缓存） |
| POST | `/api/analysis` | 支撑阻力位 + 区间涨跌幅统计 |
| GET | `/api/periods` | 支持周期列表与描述 |
| GET | `/api/watchlist` | 读取自选分组 `{groups:[{id,name,stocks}]}` |
| POST | `/api/watchlist` | 写入自选分组（规范化后落盘） |

**关键函数**:
- `norm_market_prefix(sym)` (L158): 正则把 `US/SH/SZ/HK` 前缀统一转小写，修复港股/A股 K线 404 根因（腾讯接口大小写敏感）。
- `normalize_symbol(raw, market?)` (L170): 用户输入 → 带前缀统一代码（港股补零至 5 位）。
- `market_of` / `strip_prefix` (L195/L208): 前缀解析与剥离。
- `get_klines_df(symbol, period, full)` (L264): 缓存封装，返回 `(df, source)`。
- `prewarm_cache()` (L305): 启动后台预热 `PREWARM_SYMBOLS × PREWARM_PERIODS`。
- `_default_groups()/_load_groups()/_save_groups()` (L630+): 自选分组读写，兼容旧扁平 list 自动迁移进默认分组「我的自选」。

**Pydantic 模型**: `KLinePoint`、`AnalysisRequest`、`WatchGroupModel`、`WatchlistGroupsRequest`、`SRLevel`、`RangeStat`、`AnalysisResponse`。

> 详细请求/响应字段见「数据模型与接口契约.md」。

### 2.2 `analysis.py` — 技术分析核心算法 (270 行)

- `PERIOD_PARAMS` (L28): 每周期 `half_life`/`window` 参数表（4h→30/5，1d→60/5，1w→26/4，1M→12/3，3M→8/3，6M→5/3，1Y→4/3）。
- `calculate_sr_levels(df, window=5, distance_pct=0.015, period='1d')` (L39): 自适应支撑阻力位（ATR 近似阈值 → `argrelextrema` 极值 → 指数时间衰减权重 → 支阻互换 + 临近度打分 → 各取前 3）。
- `calculate_range_stats(df, start_date, end_date)` (L188): 运行谷值/峰值法，计算 `max_rise`/`max_fall`（修正旧「全局最高点之后」退化问题）。

> 算法设计意图、五步流程、边界条件见「核心算法规格说明.md」。

### 2.3 `data_provider.py` — 数据源抽象层 (853 行，最核心)

**顶层函数**:
- `validate_klines(df, period)` (L40): 粒度校验防线，相邻日期中位数间隔判断周期是否匹配（1d≤5天、1w≤14、1M≤45、3M≤100、6M≤200、1Y≤400、4h 放行），拒绝代理退化的「季度桩」数据。
- `apply_forward_adjustment(df)` (L128): 前复权。从最新一根向前检测拆股跳变（倍率贴近 1.25…100 等常见拆股比），历史价÷倍率、量×倍率，消除拆股缺口。
- `_normalize_date(raw)` (L29): 日期格式归一化。

**类**:
- `TencentProvider` (L227): `_fetch_raw` / `_fetch_full_via_pagination`（用 `end` 参数分页回溯，突破单页约 2000 根）/ `fetch_klines` / `_get_symbol_candidates`（生成候选代码，美股加 `.OQ/.N/.AM`，A股/港股小写前缀）/ `_resample_to_4h` / `_resample_to_period`。
- `YahooProvider` (L514): 优先直连 Yahoo chart API（`range=max`，`_fetch_yahoo_direct` L623）→ 失败回退 `yfinance`；`_to_yahoo_ticker`（`sh→.SS`/`sz→.SZ`/`hk→.HK`/`us→ticker`）；`_dispatch_resample`（聚合周期先日线校验 + 前复权再聚合）。
- `DataProviderManager` (L761, 单例 `provider_manager`): 故障转移 — 美股+full 走 Yahoo（全量优先）、腾讯兜底；美股+4h 走 Yahoo（1h 更丰富）；美股日/周/月 腾讯优先；A股/港股仅腾讯（Yahoo 覆盖差）。选最长且通过 `validate_klines` 的数据；聚合周期由 provider 内前复权、其余由 manager 统一 `apply_forward_adjustment`。

`PERIOD_MAP` (L178): 周期 → 各源实际周期 + 是否重采样（4h 由 1h 重采样；3M/6M/1Y 由日线按 `QS/2QS/YS` 日历聚合）。

### 2.4 `db_cache.py` — SQLite 缓存层 (180 行)

- `init_db()`: 建 `klines` 表（PK `symbol,period,date`）+ `meta` 表 + 索引。
- `get_cached_klines(symbol, period, max_age_hours=24)`: 命中且未过期且**通过 `validate_klines`** 才返回（防污染缓存直接返回）。
- `save_klines(...)`: 先清后插（`INSERT OR REPLACE`）。
- 默认 TTL = 24h（由 `main.CACHE_TTL_HOURS` 控制）；`prewarm_cache()` 启动预热热门股。

### 2.5 `quote_provider.py` / `quote_config.py` — 实时行情 (306 + 133 行)

- `QuoteRateLimiter` (L51): 三重护盾 — 全局冷却、单 IP 滑动窗口计数、每标的最低间隔。
- `TencentQuoteProvider` (L141, 单例): `get_quotes(symbols)` 内置限流过滤，解析 `qt.gtimg.cn/q=` GBK 文本（字段下标 `_IDX_PRICE=3` 等），美股扩展时段价取字段 67；被限流标的返回 `status` 而非抛异常。
- `quote_config.py`: 所有合规红线（单标的 ≥100ms、A股 ≥2s/批量≥3s、港股 ≥2s、美股 ≥10s、盘前盘后 30–60s）、`RATE_LIMIT_COOLDOWN=30`、IP 窗口 60s/120 次、`ADAPTIVE_*`、`TICK_MS=1000` 集中此处，业务代码不硬编码。

### 2.6 测试文件

| 文件 | 行数 | 说明 |
|------|------|------|
| `test_endpoints.py` | 228 | unittest：`test_search_endpoint_*`、`test_klines_*`、`test_analysis_*`、`test_range_stats_calculation`（32.61% / -14.81%） |
| `test_adjustment.py` | 88 | NVDA 10:1、累计 40:1 拆股前复权连续性（`check()` 手动运行） |

## 四、数据流与架构关系

```
用户操作 → App.vue (UI)
  ├─ api.ts → GET /api/search ──────► main.py → 内置池 + 腾讯 smartbox 兜底
  ├─ api.ts → GET /api/klines ─────► main.py → get_klines_df
  │                                    ├─ db_cache (TTL 24h，读取仍校验)
  │                                    └─ DataProviderManager (腾讯⇄Yahoo 故障转移)
  ├─ api.ts → POST /api/analysis ──► main.py → analysis.py (SR + 区间统计)
  │                                    └─ [降级] mockEngine.ts 本地计算
  └─ quote.ts → GET /api/quote ────► quote_provider.py → 腾讯 qt.gtimg.cn/q=
                                       └─ QuoteRateLimiter 三重护盾

演示模式（/api/health 不可达自动进入）:
  handleSearch → generateMockKLines → resampleMockKLines(period) → mockEngine 分析
```

> 详细的模块调用关系、前后端数据模型对照、算法规格、降级链路见同名专项文档。

## 七、设计亮点

1. **多层降级**：数据源主备切换 → 前端 API 降级（分析走 mockEngine）→ 演示模式自动进入，全离线仍有图表。
2. **本地 SQLite 缓存**：每 (symbol, period) 全量历史持久化（TTL 24h + 启动预热），二次加载即时命中；读取仍做粒度校验防污染。
3. **前复权/拆股调整**：跨周期前向遍历检测跳变倍率，消除 NVDA 类 10:1 拆股缺口，聚合前完成避免大周期错位。
4. **实时行情合规护盾**：后端三重限流 + 前端分层刷新去重 + 自适应冷门降速；所有合规数值集中在 `quote_config.py`，前端只读取。
5. **自选分组**：用户自建分组（后端 `watchlist.json` groups 结构），市场 Tab 仅作视图过滤，不影响搜索与代码解析。
6. **符号前缀归一**：watchlist/search 返回大写前缀（USAAPL/SH600519/HK00700），后端在腾讯边界经 `norm_market_prefix` 转小写，根治港股/A股 K线 404。
