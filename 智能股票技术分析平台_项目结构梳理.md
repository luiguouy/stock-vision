# 智能股票技术分析平台 — 项目结构与代码梳理

> **项目性质**: 个人练习项目，基于 Web 的美股技术分析平台
> **当前阶段**: 第一阶段 (MVP) — 已升级多数据源 + 多周期
> **技术栈**: Vue 3.5 + TypeScript 6 + Vite 8 (前端) / Python + FastAPI (后端)
> **图表库**: TradingView Lightweight Charts v5.2.0
> **数据源**: 腾讯证券 (主) + Yahoo Finance/yfinance (备) — 自动故障转移
> **支持周期**: 日K / 周K / 月K / 季K(3M) / 半年K(6M) / 年K(1Y) — 支撑阻力位随周期自适应；季/半年/年K 由日线按日历聚合（4h 后端保留但前端隐藏）
> **前端测试模式**: 支持纯前端 Mock 数据生成 + 本地分析计算，无需后端即可调试

#### 数据模型 (Pydantic)

| 模型 | 字段 | 说明 |
|------|------|------|
| `KLinePoint` | time, open, high, low, close, volume | 单根K线数据 |
| `AnalysisRequest` | symbol, start_date, end_date, **period** | 分析请求参数 |
| `SRLevel` | price, count, type, stars, level | 支撑/阻力位 |
| `RangeStat` | pct, start, end | 涨跌幅统计 |
| `AnalysisResponse` | sr_levels, statistics, **period**, **data_source** | 分析响应 |

#### 五个 API 端点

**① `GET /api/search` — 股票搜索**

| 参数 | 说明 |
|------|------|
| `keyword` (可选) | 搜索关键词 |

逻辑：
1. 无 keyword → 返回全部预置股票池
2. 有 keyword → 匹配 symbol / name / pinyin (不区分大小写)
3. 无匹配且为纯字母 → 自动补 `us` 前缀作为美股代码返回
4. 无匹配且以 `us` 开头 → 直接返回

**② `GET /api/klines` — 多周期 K 线获取**

| 参数 | 说明 |
|------|------|
| `symbol` (必填) | 股票代码，如 `usAAPL` |
| `period` (可选) | 周期：`1d`/`1w`/`1M`/`3M`/`6M`/`1Y`（`4h` 后端仍支持，前端隐藏） |

逻辑：
1. `get_klines_df(symbol, period, full=True)` 先查本地 SQLite 缓存（TTL 24h），未命中再经 `DataProviderManager` 拉取
2. 全量历史（非 320 根截断）；季/半年/年K 由日线聚合
3. 美股代码自动尝试交易所后缀：`.OQ`(纳斯达克) → `.N`(纽交所) → `.AM`(美交所)
4. 绝不使用 Mock 假数据，拉取失败直接报 404

**③ `POST /api/analysis` — 技术分析**

请求体：
```json
{
  "symbol": "usAAPL",
  "start_date": "2024-01-01",
  "end_date": "2024-06-30"
}
```

逻辑：
1. 拉取 K 线 DataFrame
2. 调用 `calculate_sr_levels()` 计算支撑阻力位
3. 调用 `calculate_range_stats()` 计算区间涨跌统计
4. 返回合并结果

#### 腾讯数据拉取函数

**`_fetch_tencent_direct(symbol)`**
- 请求 URL: `https://web.ifzq.gtimg.cn/appstock/app/fqkline/get?param={symbol},day,,,320,qfq`
- 带 User-Agent 伪装浏览器
- 解析 `qfqday`（前复权）或 `day` 字段
- 腾讯K线格式: `['日期', '开', '收', '高', '低', '成交量']`

**`fetch_kline_df(symbol)`**
- 已含后缀 → 直接查询
- `us` 开头 → 依次尝试 `.OQ` / `.N` / `.AM`
- 兜底直接尝试原始 symbol

### 2.2 `analysis.py` — 技术分析核心算法

#### `calculate_sr_levels(df, window=5)` — 支撑/阻力位算法

**五步算法流程：**

```
Step 1: 自适应聚类阈值
  ├── 计算最近 20 天日均波幅 (ATR 近似)
  ├── 安全边界: [0.6%, 2.5%] × 最新价
  └── abs_threshold = max(0.6%, min(ATR×0.8, 2.5%))

Step 2: 寻找局部极值点
  ├── scipy.signal.argrelextrema(high, np.greater_equal, order=5) → 局部最高
  └── scipy.signal.argrelextrema(low, np.less_equal, order=5)     → 局部最低

Step 3: 价格合并 + 时间衰减权重
  ├── 按价格排序，相近价格（阈值内）合并
  ├── 半衰期 60 天的指数衰减: weight = 0.5^(days_ago / 60)
  ├── 合并时动态更新加权平均价格
  └── 今天=1.0, 60天前=0.5, 120天前=0.25

Step 4: 支阻互换 + 临近度打分
  ├── 以最新收盘价为基准
  ├── 低于最新价 → 支撑 (support)
  ├── 高于最新价 → 阻力 (resistance)
  ├── 临近度因子: 1 / (1 + 8 × 距离百分比)
  └── 综合得分: weight × proximity_factor

Step 5: 排序输出 Top3
  ├── 支撑和阻力各取 Score 前 3
  └── 星级: weight≥3.5 → Strong(5★), ≥2.0 → Medium(3★), 其他 → Weak(2★)
```

#### `calculate_range_stats(df, start_date, end_date)` — 区间涨跌统计

**最大上涨：**
1. 筛选日期区间内数据
2. 找到区间内 `low` 最低点
3. 在该最低点**之后**寻找 `high` 最高点
4. 涨幅 = `(高 - 低) / 低 × 100`

**最大下跌：**
1. 找到区间内 `high` 最高点
2. 在该最高点**之后**寻找 `low` 最低点
3. 跌幅 = `(低 - 高) / 高 × 100`

输出：涨跌幅百分比（保留 2 位小数）+ 起止日期。

### 3.3 `src/utils/api.ts` — API 封装层

封装三个后端接口调用 + Mock 数据生成，定义 TypeScript 接口类型。

#### TypeScript 接口定义

```typescript
interface StockInfo { symbol, name, pinyin? }
interface KLinePoint { time, open, high, low, close, volume }
interface SRLevel { price, count, type: 'support'|'resistance', stars, level: 'Weak'|'Medium'|'Strong' }
interface RangeStatPoint { pct, start, end }
interface RangeStats { max_rise: RangeStatPoint, max_fall: RangeStatPoint }
interface AnalysisResponse { sr_levels: SRLevel[], statistics: RangeStats }
```

#### 四个函数

**`searchStocks(keyword)`**
- `GET /api/search?keyword=...`
- 失败抛出异常

**`getKLines(symbol)`**
- `GET /api/klines?symbol=...`
- **降级机制**: 后端失败时，前端直接请求腾讯接口（使用新版 `newstrend/new_chart/query` 接口，绕过后端）

**`getStockAnalysis(symbol, startDate, endDate, klineCache?)`**
- `POST /api/analysis`
- **降级机制**: 后端不可用时，调用 `mockEngine.ts` 在前端本地计算
- 可传入已缓存的 K 线数据避免重复请求

**`generateMockKLines(symbol, days, basePrice)`** *(新增)*
- 生成指定天数的模拟K线数据，用于前端独立调试
- 随机游走算法生成价格，不同股票有不同波动率
- 自动跳过周末，成交量随机生成

### 3.4 `src/utils/mockEngine.ts` — 前端降级计算引擎

后端不可用时的**纯前端计算引擎**，将 `analysis.py` 的算法完整移植到 TypeScript。

> 注: 这不是 Mock 假数据，而是纯数学计算的降级方案。

#### `findExtrema(data, windowSize=5)`
- 滑动窗口寻找局部最高/最低点
- 返回 `{price, idx}[]`

#### `calculateSRLevels(data, windowSize=5)`
- 与后端 `calculate_sr_levels()` 完全一致的 5 步算法
- 自适应阈值、时间衰减权重、支阻互换、临近度打分、Top3 输出

#### `calculateRangeStats(data, sDate, eDate)`
- 与后端 `calculate_range_stats()` 完全一致
- 最大上涨: 最低点 → 之后最高点
- 最大下跌: 最高点 → 之后最低点

## 四、数据流与架构关系

### 4.1 整体数据流

```
用户操作
  │
  ▼
App.vue (UI 交互)
  │
  ├──► api.ts → GET /api/search ──────► main.py → 本地股票池模糊匹配
  │
  ├──► api.ts → GET /api/klines ─────► main.py → 腾讯接口拉取日K
  │                                    │
  │                                    └── [降级] api.ts 直连腾讯接口
  │
  └──► api.ts → POST /api/analysis ──► main.py
                                        │
                                        ├──► analysis.py
                                        │     ├── calculate_sr_levels()
                                        │     └── calculate_range_stats()
                                        │
                                        └── [降级] mockEngine.ts 本地计算
```

### 4.2 测试模式数据流

```
用户开启测试模式（默认开启）
  │
  ▼
handleSearch()
  │
  ├──► generateMockKLines(symbol) → 随机游走生成 320 根模拟日K
  │
  └──► runAnalysis()
        │
        ├──► mockEngine.calculateSRLevels(klines) → 本地计算支撑阻力位
        └──► mockEngine.calculateRangeStats(klines, start, end) → 本地计算区间统计
```

### 4.3 核心算法数据流

```
K线数据 (320根日K)
  │
  ▼
Step 1: argrelextrema 找局部极值 (window=5)
  │
  ▼
Step 2: 自适应阈值聚类 (ATR近似, 0.6%~2.5%)
  │
  ▼
Step 3: 时间衰减权重累加 (半衰期60天)
  │
  ▼
Step 4: 支阻互换 + 临近度综合打分
  │        score = weight × 1/(1 + 8×dist_pct)
  │
  ▼
Step 5: 排序输出 Top3 (支撑 + 阻力各3个)
         Strong(5★) / Medium(3★) / Weak(2★)
```

## 七、设计亮点

1. **双层降级机制**: 前端 `api.ts` 在后端不可用时可直连腾讯接口 (K线) 或本地计算 (分析)，保证可用性
2. **测试模式**: 前端内置 Mock 数据生成引擎，随机游走算法生成逼真假K线，无需后端即可调试全部功能
3. **支阻互换机制**: 支撑/阻力不是固定的，而是根据最新价动态判定，更符合实战逻辑
4. **时间衰减权重**: 半衰期 60 天，近期极值权重高，远期递减，避免过时支撑阻力干扰
5. **自适应聚类阈值**: 基于近期 ATR 自动调整合并距离，适配不同价位和波动率的股票
6. **前后端算法一致性**: `mockEngine.ts` 是 `analysis.py` 的 TypeScript 完整移植，降级计算结果一致
7. **三栏可拖拽布局**: 左右面板宽度可拖拽调节（范围 200~页面1/3），图表实时跟随缩放（`autoSize` + 双重 RAF 显式同步）
8. **SR 双向联动**: 悬停 SR 列表项时图表 priceLine 加粗高亮，点击时闪烁提示，图表与分析面板深度联动
9. **CSS 变量主题系统**: `:root` + `.dark` 统一管理全局色板，支持未来深色模式完整切换
10. **键盘交互增强**: 搜索框 ↑↓ Enter Esc 导航、Ctrl+K 全局搜索、R 快捷重算，提升操作效率
